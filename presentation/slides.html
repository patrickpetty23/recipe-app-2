Section 01

The Product

Six screens. Infinite recipes. Let's walk through how it works.

01
Feature 01 — Capture

Add any recipe in seconds

📷

Camera Scan

Point at a cookbook, magazine, handwritten card. GPT-4o extracts every ingredient and step — even from messy handwriting.

🔗

URL Import

Paste any recipe URL. The AI strips ads, pop-ups, and life stories — leaving only the actual recipe.

💬

Conversational Chat

Describe what you want to cook. The AI builds a full recipe from scratch or answers any cooking question in real time.

⚡ Average recipe capture time: 8 seconds — vs. 3–5 minutes of manual entry in competitor apps

Feature 02 — Cooking

A guided kitchen assistant in your pocket

🎨

Auto-generated visuals

Every recipe gets a professional food photo thumbnail and step-by-step illustrations — generated automatically by DALL-E 3, no camera required.

🔊

Voice-guided cooking mode

Hands-free, text-to-speech narration reads each step aloud. Swipe or tap to advance. Screen stays on. Built-in countdown timers.

⚖️

Smart portion scaling

Adjust servings and every ingredient quantity updates instantly. Scales up for a dinner party or down for one.

✨

"Make it Lighter"

One tap generates a healthier version of any recipe — AI swaps heavy ingredients, shows calorie delta, sends straight to the editor.

Feature 03 — Nutrition

Calorie tracking that actually works

The fundamental problem with calorie trackers: users have to manually log every meal. Nobody does this for more than a week.

"What if nutrition tracking was a side effect of cooking — not a separate chore?"

Because users are already cooking in Mise, they tap "Log Meal" once. GPT-4o estimated the macros automatically when the recipe was saved. Zero manual entry.

Technical Differentiation

Built to be fast, private, and smart

🗄️

Offline-first architecture

Every recipe, ingredient, and cook log lives in a local SQLite database with WAL mode. Zero latency, zero data fees, works in the kitchen with no signal.

⚡

Non-blocking AI pipeline

Saving a recipe is instant. Nutrition estimation, thumbnail generation, and step illustrations all fire in the background — user never waits for AI.

🔒

Privacy by design

No user data leaves the device except anonymised API calls to OpenAI. No account required on free tier. No tracking SDKs.

🚀

Cross-platform from day one

Single React Native / Expo codebase ships to both iOS and Android simultaneously, halving ongoing development cost vs. native.

Appendix
Appendix — Technical Stack

Full Technology Overview

Frontend

  • React Native (Expo SDK 54)
  • Expo Router v6 (file-based routing)
  • expo-linear-gradient (hero images)
  • expo-speech (TTS narration)
  • expo-keep-awake (cooking mode)
  • PanResponder (swipe gestures)
  • Animated API (smooth transitions)

Data & Storage

  • expo-sqlite v16 (WAL mode)
  • FK cascade constraints
  • Parameterised queries (SQL injection safe)
  • Offline-first — no cloud DB
  • cook_log + recipe_nutrition tables
  • Background migrations on boot

AI / Cloud

  • GPT-4o — recipe extraction, chat, nutrition
  • DALL-E 3 — hero thumbnails (1792×1024)
  • DALL-E 2 — step illustrations (parallel)
  • JSON mode for structured output
  • 45s / 60s timeout guards
  • Non-blocking background pipeline
