# Founding Hypothesis

## Core Hypothesis

If we reduce recipe-to-shopping-list creation time by at least 50% for home cooks using any recipe source (URL, photo, text, PDF), users will adopt the app weekly even if AI extraction is not perfect, as long as edit corrections stay under 10 seconds for common errors.

## Why This Hypothesis

1. The workflow is frequent (weekly meal planning).
2. Existing alternatives are manual and repetitive.
3. Users prioritize speed and reliability over feature complexity.

## Assumptions

1. Users tolerate minor AI mistakes if corrections are fast.
2. GPT-4o extraction quality is sufficient for a first-pass list on common inputs.
3. Checklist persistence has higher user value than advanced AI features.
4. Walmart cart integration creates a differentiated last-mile step no competitor offers.

## Observable Predictions

1. Median time-to-saved-recipe under 60 seconds (including editor review).
2. Users report app flow as faster than Notes/Reminders baseline.
3. At least half of tested users indicate weekly reuse intent.
4. Users who see Walmart cart integration express interest in using it.

## Invalidating Conditions

The founding hypothesis is invalid if:

1. App workflow is not faster than manual baseline.
2. Users do not trust AI output despite the structured editor.
3. Repeat intent is weak after first session.
4. Walmart cart adds complexity without perceived value.

## Evolution from Midterm

The midterm hypothesis focused narrowly on camera scanning of cookbooks and OCR speed. Building revealed:

- **URL import is 3x more popular than camera** — the hypothesis holds, but the primary input method changed.
- **The product grew beyond recipe-to-list** — users wanted nutrition tracking, cooking guidance, and meal planning. The hypothesis expanded from "faster list creation" to "full cooking lifecycle companion."
- **Walmart cart was validated unprompted** by Thomas before seeing the feature, confirming the differentiation assumption.
- **Nutrition tracker emerged from user feedback** (not the PRD), confirming that listening to users produces features the hypothesis alone wouldn't predict.

## Current Status

- Core hypothesis validated across 10 user sessions (3 midterm + 7 final).
- Speed advantage confirmed: URL import ~8s vs. minutes of manual entry.
- Trust validated: structured per-field editor catches parsing errors effectively.
- Reuse intent: all 7 final participants said they would use the app regularly.
- Walmart validation: 6/7 final users endorsed the cart integration.
